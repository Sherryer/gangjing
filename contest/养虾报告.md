## 三、它是什么

**杠精虾** — 多模态批判性思维 Review 引擎

"杠精"在这里不是网络喷子，而是**建设性挑刺型专家**。没有情绪包袱，没有人情压力，只有一个目标：把问题在交付前找出来。

### 核心能力全景

```
输入层（丢什么都能杠）
├── 代码文件      → 安全漏洞 / 性能问题 / 设计缺陷
├── 商业方案 / 文档 → 逻辑漏洞 / 数据假设 / 竞争盲区
├── 网页 URL      → 一键抓取正文，即时 review
├── PDF 文档      → 解析全文，结构化输出问题
└── 视频文件      → ASR 转录 + 逐段审查（文画一致性 / 事实核查）

引擎层（怎么杠）
├── 单虾模式      → 直接 review，30 秒出报告
└── 三虾互杠模式  → ProposalShrimp 出初稿
                   CriticShrimp + DevilShrimp 并发评审
                   自动迭代至 P0 清零（最多 3 轮）

输出层（杠完给什么）
├── 结构化 Review Report（P0-P3 分级，精准定位，可执行建议）
├── Rich 终端彩色报告（P0 红色 / P1 橙色 / 通过绿色，一眼看完）
└── HTML 报告一键导出（自包含，离线可用，直接交给甲方）
```

### 杠精等级三档

| 等级 | 适用场景 | 风格 |
|------|---------|------|
| 🟢 Level 1 温柔杠 | 草稿阶段快速过一遍 | 挑关键问题，留余地 |
| 🟡 Level 2 正常杠 | 日常 review 标准档 | 全面扫描，直球指出 |
| 🔴 Level 3 魔鬼杠 | 上线前 / 重要交付终审 | 无死角，模拟最刁钻的挑刺者 |

## 四、为什么不一样

很多人第一反应：直接问 GPT 不就行了？

| 维度 | 直接问 GPT | 杠精虾 |
|------|-----------|--------|

| 问题评级 | 没有，全是建议 | P0-P3 量化，知道哪个必须改 |

| 输入范围 | 主要是文本粘贴 | 代码文件 / URL / PDF / 视频 |
| 主动性 | 你问它才说 | 心跳巡检，主动找你报告 |
| 场景适配 | 通用 | 代码 / 商业 / 内容 领域专业模块 |

核心差异只有一句话：**GPT 帮你优化，杠精虾帮你找死。**

## 七、技术架构

### 四层分层设计

```
┌─────────────────────────────────────────────┐
│              输入适配层                      │
│   文本 / 代码 / URL / PDF / 视频 → 标准文本  │
├─────────────────────────────────────────────┤
│              批判引擎层                      │
│   单虾 Review / 三虾互杠 / 收敛检测          │
├─────────────────────────────────────────────┤
│              领域专业层（可插拔）             │
│   代码模块 / 商业模块 / 内容模块 / 自定义     │
├─────────────────────────────────────────────┤
│              标准输出层                      │
│   P0-P3 结构化报告 / Rich 终端 / HTML 导出   │
└─────────────────────────────────────────────┘
```

**技术亮点：**
- 多 Agent 异步并发（asyncio.gather），CriticShrimp + DevilShrimp 同时评审，不等待
- P0 计数收敛算法，自动判断是否继续迭代，内置防无限循环保护
- 模型层完全可插拔，支持 DeepSeek / Claude / OpenAI / Qwen / 本地模型，一行配置切换
- 本地模型支持微调，垂直场景适配专属评审偏好（代码规范 / 行业术语 / 企业风格）
- HTML 报告自包含，内嵌 CSS + JS，离线可用，零依赖
