# **二、对养虾和Agent的深度思考**

我们观察到一个现象：不管是写代码、写方案、写文章还是做决策，**人最难发现自己的盲点**。团队里最有价值的人往往不是那个执行最快的，而是那个能在关键时刻说出"等一下，这里有问题"的人。

但现实中，这种角色极度稀缺——它要求专业能力、批判思维、直言不讳的性格三者兼备。大多数团队要么没有这样的人，要么有但碍于人情世故不敢说真话。

**我们的核心洞察：Agent 天然适合扮演这个角色。** 它没有情绪、没有人情压力、不会因为"怕得罪人"而放水，只要规则设计得当，它可以成为每个人身边 24 小时待命的"高水平审阅者"。

这不是一个新鲜概念——但做好极难。简单写一段"请帮我找问题"的 prompt 谁都会，真正的挑战在于：如何让它**杠得准**（不漏关键问题）、**杠得有用**（每条批评都附带可执行的改进方案）、**杠得舒服**（专业直白但不让人想关掉）、**杠得广**（不只是代码，什么都能杠）。

这就是我们养"杠精虾"的出发点。

# **三、本次养殖虾类品类及核心特性**

## 品类：杠精虾 —— 多模态批判性思维引擎

"杠精"在这里不是网络喷子，而是**建设性挑刺型专家**。本质是高水平审阅者 / 红队 / 批判性思维助手。

### 核心特性

**1. 分层架构，不是一个 prompt**

杠精虾不是"一段厉害的提示词"，而是一套完整的分层架构：通用批判引擎 + 可插拔的领域专业模块。通用层掌握"怎么杠"的方法论，领域层提供"杠什么"的专业知识。新增一个领域，只需插入一个新模块，零改动核心。

**2. 多模态输入，不只是文本**

丢文本它能杠，丢代码它能杠，丢视频它也能杠。通过语音识别 + 关键帧分析，它能指出"你第 2 分 15 秒说的数据和第 4 分 30 秒的画面自相矛盾"。未来还支持 PDF、图片、URL 直接输入。

**3. 杠精等级三档可调**

不是永远全力杠到底。三档等级从"温柔过一遍"到"魔鬼终审"，适配不同场景。快速看一眼草案用 Level 1，上线前终审用 Level 3。随时一句话切换。

**4. 标准化输出，不是散文式评价**

每次审查输出结构化的 Review Report：问题按严重程度分级（P0-P3）、按专业维度分类、精准定位到具体位置、每条附带可执行的修复建议。不是模糊地说"这里不太好"，而是精确地说"第 23 行有内存泄漏，因为 XX，建议改为 YY"。

**5. 多虾互杠，逼出最优解**

支持多 Agent 协作模式：方案虾出方案 → 杠精虾挑刺 → 对立虾质疑根本假设 → 迭代至无明显硬伤。三个不同视角互相碰撞，避免单一模型的思维盲区。内置防无限循环机制。

**6. 心跳主动巡检**

不只是"你丢给它它才审"，杠精虾可以定期主动审查最近的产出。第二天早上打开电脑，昨天写的代码已经被 review 好了，报告安静地等在那里。带智能降噪——同一个问题不重复唠叨。

# **五、解决业务场景的实际问题**

### 场景一：深夜赶代码，没人帮你 Review

> 凌晨 2 点终于写完新功能，队友都睡了。直接提交？心里没底。明天等 review？deadline 过了。

**杠精虾方案**：丢进去，30 秒出 Review Report，P0 问题一目了然。该改的改完再提交，安心睡觉。

### 场景二：创业方案写了 20 页，自我感觉良好

> 写了一份完美的商业计划书，觉得逻辑严密、数据充分。拿去融资被投资人灵魂拷问，一问三不知。

**杠精虾方案**：提前让杠精虾做一轮"投资人模拟拷问"。它会质疑你的市场规模假设、追问你的用户增长数据来源、指出你忽略的竞争对手。见投资人之前先把该补的功课补了。

### 场景三：科普视频要发了，不确定有没有硬伤

> 花了一周做的科普视频，怕里面有事实错误被弹幕喷。自己反复看了三遍，但当局者迷。

**杠精虾方案**：视频丢进去，自动转录 + 逐段审查。"3 分 40 秒的说法与最新研究结论不符""数据图表的 Y 轴截断可能引起误解"——发布前把硬伤清零。

### 场景四：团队方案讨论，大家都在"好好好"

> 方案评审会上，碍于面子没人提反对意见，最后执行时踩了一堆坑。

**杠精虾方案**：启动多虾互杠模式，让三只虾从不同角度碰撞。它没有人情负担，该说的都会说。方案经过多视角打磨后再拿到会上讨论，质量完全不同。

# **六、提效目标**

| 指标 | 当前（无杠精虾） | 目标（有杠精虾） | 提升 |
|------|----------------|----------------|------|
| **单次 Code Review 耗时** | 人工 15-30 分钟 | 30 秒出报告 | **30-60x** |
| **问题发现率** | 人工 review 平均漏检 20-40% | 系统化多维度扫描，漏检率 < 10% | **显著提升** |
| **方案迭代轮次** | 平均 3-5 轮（等人反馈） | 多虾互杠 2-3 轮自动完成，人只做终审 | **节省 60%+ 等待时间** |
| **重要文档发布前自检** | "算了来不及了先发吧" | 1 分钟内完成多维度自检 | **零成本兜底** |
| **审查覆盖率** | 取决于 reviewer 是否有空 | 心跳巡检保证 100% 覆盖 | **从看运气到全覆盖** |

**核心提效逻辑：** 不是替代人类的判断力，而是把"找问题"这个最耗时、最容易疲劳、最受人情影响的环节自动化，让人专注于"做决策"。

# **七、具体实施方案**

### 整体架构

四层分层设计：**输入适配层 → 批判引擎层 → 领域专业层 → 标准输出层**。每一层职责清晰、互不耦合，新增领域或输入格式时只需扩展对应层，不影响其他部分。

### 实施节奏

| 阶段 | 内容 | 交付物 |
|------|------|--------|
| **Phase 1** | 单虾文本杠精 MVP | 支持代码/商业方案/文案三个领域的结构化 review |
| **Phase 2** | 多虾互杠协作 | 三虾工作流 + 自动迭代 + 防循环保护 |
| **Phase 3** | 心跳 + 多模态 | 定时主动巡检 + 视频/PDF/URL 输入支持 |
| **Phase 4** | 打磨 + 演示 | 杀手级案例准备 + 比赛材料整理 |

### 技术亮点（不展开细节）

- 通用批判方法论与领域专业知识解耦，可无限扩展领域
- 三档杠精等级适配不同严肃程度的场景
- 多 Agent 多视角互杠机制，内置收敛保护
- 心跳巡检 + 智能降噪，不骚扰只兜底
- 多模态输入流水线，视频也能杠
