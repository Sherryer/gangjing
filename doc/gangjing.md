# “杠精” 虾

“杠精”在这里不是故意找茬、阴阳怪气那种网络喷子，而是**建设性挑刺型**：专门帮你发现计划/方案/代码/文章/决策里的逻辑漏洞、考虑不周的地方、潜在风险、优化盲点，然后直球指出问题、给出改进建议。  
本质是**高水平审阅者 / 红队 / 批判性思维助手**，不是为了杠而杠，而是为了让结果更robust、更靠谱。

OpenClaw（小龙虾）超级适合养成这种“杠精虾”，因为它：

- 支持**持久记忆**（SOUL.md / IDENTITY.md / MEMORY.md），可以长期记住你的风格偏好：“我就是你的杠精模式，永远先找问题，再给方案”
- 支持**多智能体**（multi-agent），可以专门搞一个“Critic Agent / Reviewer Agent / 红队虾”专门负责挑刺
- 支持**自定义灵魂/身份** + **Skills**，可以写死规则让它每次输出都先批判后建设
- 支持**Heartbeat**（心跳任务），可以让它主动审阅你昨天的输出/计划

### 如何养一只专业的“杠精虾”？（2026年3月主流玩法，按难度排序）

1. **最简单（云端/大厂版，几分钟搞定）**  
   用 Kimi Claw、MaxClaw、Doubao Claw 等云版本，直接在**系统提示 / 人格设定**里写死杠精规则：  
   示例提示（复制粘贴到人格描述或每次对话开头）：
   
   ```
   你现在是我的“杠精虾”——专业、犀利但不带情绪的审阅专家。
   规则（铁律，永远严格遵守）：
   1. 收到任何方案、代码、文章、计划、想法，先**不表扬**，直接进入“找茬模式”：
      - 逻辑漏洞？矛盾点？假设不成立的地方？
      - 边界case / 极端情况没考虑？
      - 风险 / 副作用 / 法律/道德/安全隐患？
      - 更高效/更简洁/更优雅的替代方案？
   2. 每条批评必须：
      - 指出具体问题 + 为什么是问题（讲道理）
      - 给出可执行的改进建议
   3. 挑完刺后，最后一段才可以总结优点 + 优化后的完整版本（如果有）
   4. 语气：专业、直白、像资深工程师 code review，不要阴阳、不要“哈哈你这不行哦”
   5. 如果实在没问题，才可以说“目前看来没明显硬伤，但建议关注XX”
   
   从现在开始，所有对话都强制进入这个模式，除非我说“关闭杠精模式”。
   ```
   
   → 这样你的虾基本就杠精化了，随时丢方案给它，它先给你来一顿“灵魂拷问”。
2. **进阶本地/自部署版（推荐，控制力最强）**
- 在 `~/.openclaw/workspace/SOUL.md` 或 `IDENTITY.md` 里写上上面那套铁律，作为虾的永久人格。
- 再新建一个专用 Agent（multi-agent 神器）：  
  在 `AGENTS.md` 里加一行：
  
  ```
  CriticShrimp: 杠精虾 / Reviewer Agent。永远先审阅、找问题、提风险，再给建设性方案。使用模型：deepseek-r1 或 qwen-max（逻辑强）。禁止直接执行破坏性操作。
  ```
  
  然后你发任务时可以指定：`@CriticShrimp 帮我 review 这段代码/这个商业计划`
- **终极神器**：装社区的 review/critique 系 Skills（ClawHub 搜 “review”“critic”“audit”“redteam”），2026年已经有几百个，比如：
  - PR Review Agent（代码审阅专用）
  - Risk Auditor Skill（风险扫描）
  - Debate Mode Skill（正反方辩论，相当于自己跟自己杠）
3. **硬核玩法：虾群杠精模式（多虾互杠，逼出最优解）**  
   养 3 只虾：
- 主虾：Proposal Shrimp（负责出方案）
- 杠精虾1：Critic Shrimp（只挑刺）
- 杠精虾2：Devil’s Advocate Shrimp（故意站在对立面抬杠，逼出盲点）
   
   工作流示例（写到 HEARTBEAT.md 或手动触发）：
   1. Proposal Shrimp 先出一版方案
   2. Critic Shrimp + Devil’s Advocate Shrimp 同时 review
   3. Proposal Shrimp 根据反馈迭代 v2
   4. 循环 2-3 轮，直到 Critic 们说“没明显硬伤”
   
   这招在写论文、做产品方案、debug 复杂系统时特别猛，很多“养虾人”都靠这个把输出质量拉到接近人类 senior 水平。

### 注意事项（很重要！）

- 杠精模式开太狠，虾容易“过于保守”或“无限循环找问题”，建议加一句铁律：`挑刺控制在3-8条，超过就总结最关键的`
- 模型选逻辑强的：国内推荐 DeepSeek R1、Qwen3-72B-Instruct、Claude-4系列；贵点但杠得准
- 先用非关键任务测试，别直接让它 review 你写给老板的 PPT 或发生产代码
